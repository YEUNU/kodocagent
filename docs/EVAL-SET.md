# 문서 편집 의도 충실도 검증 셋 (0.6.x)

> 목적: "에이전트가 문서를 **100% 사용자 의도대로** 수정하는가"를 측정한다.
> 단순 편집이 아니라 **문서 구조 이해 + 정합성 유지**까지 본다.
> 작성 2026-06-17. 프롬프트 원안 30 + 실무형 5 = **35종**.

## 0. 전제 — 검증 문서

레포에는 **커밋된 실문서 fixture가 없다**(테스트는 `markdownToHwpx`로 합성). 따라서 이 셋은 두 부류로 나뉜다.

| 코드 | 문서 | 생성 방법 | 비고 |
|---|---|---|---|
| **F1** | 사업계획 보고서(.hwpx) | 합성(markdownToHwpx) | 제목/본문/날짜·금액·기관명·오탈자·참고목록 |
| **F2** | 예산·실적 표 보고서(.hwpx) | 합성 | 단순 격자표 + 본문 수치(일부 불일치 삽입) |
| **F3** | 신청서 양식(.hwpx) | 합성 + XML 병합셀 주입 | 라벨/값 2열·병합 헤더·빈칸 |
| **F4** | 양식개체 문서(.hwpx) | 합성 + XML 양식개체 주입 | 누름틀·편집상자·콤보·체크 |
| **F5** | 개인정보 안내문(.hwpx/.md) | 합성 | 주민/전화/이메일/카드번호 |
| **F6** | **실 공공문서(.hwp/.hwpx)** | **사용자 제공 필요** | 복합표(중첩·병합)·머리말/꼬리말·쪽번호·각주·목차·이미지(로고/차트/조직도)·개요번호·그림·상호참조·스타일 |

> **F6가 핵심 한계**: 머리말/꼬리말·각주·목차·이미지·스타일·복합표·상호참조·자동 개요번호는 `markdownToHwpx`로 **합성 불가**다. 하드/실무형 프롬프트 대부분이 F6에 의존하므로, **실 공공문서 2~3건을 확보해야 완전한 벤치마크가 된다.**

## 1. 판정 범례 & 보유 도구

- **✅ 지원**: 현재 도구로 의도대로 수정 가능
- **⚠️ 부분**: 일부만/특정 조건/에이전트 추론 의존(자동 보장 없음)
- **❌ 미지원**: 현재 도구로 불가 — 새 기능 필요

보유 편집 도구: `propose_edit`(.hwp 제자리·.hwpx 무손실 텍스트), `propose_find_replace`(본문·표·머리말 텍스트 전역 치환), `propose_cell_edit`(표 셀 좌표/라벨·병합 보존·빈칸), `propose_table_structure`(행·열·병합, 병합 가로지름 거부), `list_form_objects`/`propose_form_object`(양식개체 5종), `propose_redact_pii`/`scan_pii`(PII), `propose_sheet_edit`(xlsx), `find_in_document`, `compare_documents`, `list_backups`/`restore_backup`.
**없는 능력**: 문단/제목 **스타일 시스템**, 글꼴·줄간격·굵게 등 **문자/문단 서식**, **머리말/꼬리말·쪽번호 레이아웃**, **각주**, **목차 자동 갱신**, **자동 개요번호**, **상호참조(그림/표 번호 ↔ 본문)**, **이미지/도형/조직도**, **중첩표 안쪽 셀**, 문서 전역 **자동 정합성 보장**.

## 6. 종합 — "100% 의도대로"는 현재 어디까지?

**가능성 집계(35종)**: ✅ 약 6 · ⚠️ 약 19 · ❌(또는 ❌에 가까운 ⚠️) 약 10.

- **잘 되는 영역(✅/안정 ⚠️)**: 텍스트 편집, 전역 용어/명칭 치환, 표 셀 값(병합 보존), 날짜·금액 정규화, PII 비식별, 법령 MCP 연동, 행·열 구조 편집(최상위·비병합). → **쉬움 티어와 텍스트/표 중심 중간 티어는 상당 부분 충족**.
- **막히는 영역(❌)** = 0.6.x에서 새로 만들어야 할 능력:
  1. **문단/제목 스타일 시스템**(#2, C) — 동급 제목 일관화
  2. **문자/문단 서식**(굵게·글꼴·줄간격, #5·#8)
  3. **머리말/꼬리말·쪽번호 레이아웃**(#19, C)
  4. **각주 편집**(#20, #26, C)
  5. **목차 자동 갱신 / 자동 개요번호**(#10, #14)
  6. **상호참조(그림·표 번호 ↔ 본문)**(#18, #30)
  7. **이미지/도형/조직도**(#22, B)
  8. **중첩표(복합표) 안쪽 셀**(#25)
  9. **전역 정합성 보장 + 강제 용어집**(#13·#21·#26·#30)

> 결론: **순수 텍스트·표 값 수정은 의도 충실도가 높지만**, 한글 공문서의 "정비/형식/구조/정합" 요구(하드·실무형 대부분)는 **현재 도구로 100% 달성 불가**. 위 9개가 0.6.x의 기능 로드맵 후보다.
