# GUI 설계 — kodocagent 데스크톱 (Electron)

> 2026-06-18 기준(npm CLI 0.7.1). CLI에 구현된 기능과 "비전문가도 안심하고 쓰는 문서 AI 챗봇" 목표를 바탕으로 GUI가 가져야 할 모습을 정의한다.
> ROADMAP상 GUI는 1.0(CLI)과 **독립된 0.x 트랙**. 핵심 로직(`@kodocagent/core`)은 CLI/GUI 공용이며, GUI는 UI 레이어만 추가한다.

---

## 1. 설계 원칙 (왜 GUI인가)

대상 사용자는 **한글(.hwp/.hwpx)·오피스 문서를 다루는 한국의 비전문가**다. 이들이 AI 문서 도구에서 신뢰를 잃는 지점은 일관된다: *"내 파일이 망가지나? · 취소되나? · 뭘 승인하는 건가? · AI가 멋대로 지어내나?"*. GUI의 존재 이유는 **이 불안을 시각적으로 해소**하는 것이다.

1. **신뢰 우선(Trust-first)** — 원본은 승인 전까지 절대 안 바뀐다는 사실을 항상 보이게. 모든 변경은 미리보기 → 승인 → 적용. 되돌리기는 한 클릭.
2. **결과를 보여준다(Show, don't tell)** — 마크다운 요약이 아니라 **실제 문서를 렌더링**해 보여준다. 변경 전/후를 시각적 diff로. (CLI가 한컴 뷰어 육안 검증으로 버그를 잡았듯, GUI는 그 "실제 결과 확인"을 사용자 손에 쥐여준다.)
3. **자연어 + 한 클릭(Conversational + quick actions)** — 채팅으로 자유롭게, 동시에 자주 쓰는 작업(요약·검토·개인정보 가리기·내보내기)은 버튼으로.
4. **정직(Honesty)** — 능력·한계를 그대로 노출. 못 하는 작업은 회색 처리/안내. 약한 모델도 목적을 달성하도록 자가검증·thrash 억제가 백그라운드에서 작동(사용자는 "검증 중" 정도만 인지).
5. **CLI와 동일한 엔진** — `AgentSession`·`ApprovalHandler`·이벤트 스트림·도구 레지스트리를 그대로 사용. GUI 고유 로직은 렌더링·상호작용뿐(중복 구현 금지).

## 2. 현재 스캐폴드 (이미 있는 것)

`packages/gui` (Electron + React + Vite):
- `main/agent-bridge.ts` — `AgentBridge`가 `AgentSession`을 띄우고 이벤트를 렌더러로 IPC 전달, 승인 pending map으로 다이얼로그와 연동.
- `renderer/App.tsx` + 컴포넌트: `ChatView`(스트리밍 대화·툴콜 표시), `ApprovalDialog`(승인), `StatusBar`(모델/상태), `Composer`(입력), cwd 변경.

→ **채팅 + 2단계 승인의 최소 골격은 동작**한다. 아래는 이 위에 쌓을 목표 정의다.

## 3. 화면 레이아웃 (3-pane 워크스페이스)

```
┌─────────────────────────────────────────────────────────────────────┐
│  ⬤ kodocagent          작업 폴더: ~/문서/계약   [모델 ▾]  [컨텍스트 38%]│  ← 상단바
├───────────────┬───────────────────────────────┬─────────────────────┤
│  파일 패널     │   문서 미리보기 패널            │   대화 패널          │
│ (좌, 접힘 가능)│   (중앙 — 실제 렌더)           │   (우)              │
│               │                               │                     │
│ 📂 드래그&드롭 │  ┌─────────────────────────┐  │  나: 표 금액에 콤마  │
│  open         │  │ 2026년 예산·실적 보고서  │  │     넣어줘           │
│               │  │  ┌──────┬──────┐         │  │  ─────────────────  │
│ 📄 report.hwpx│  │  │ 항목 │ 금액 │ ← 변경  │  │  ⚙ read_document    │
│ 📄 계약서.hwp │  │  │ 제작 │1,500,000│      │  │  ⚙ propose_cell_edit│
│               │  │  └──────┴──────┘         │  │  ✓ 검증 완료        │
│ ⏱ 되돌리기     │  └─────────────────────────┘  │                     │
│  · 14:02 콤마  │   [원본] [변경본] [diff] 탭   │  [요약][검토][PII][⇩]│  ← 빠른 작업
└───────────────┴───────────────────────────────┴─────────────────────┘
│  입력: "이 표의 합계를 다시 계산해줘"                          [전송] │  ← Composer
└─────────────────────────────────────────────────────────────────────┘
```

- **좌(파일/타임라인)**: 작업 폴더 파일 목록(`list_files`), 드래그&드롭 열기, **되돌리기 타임라인**(`list_backups`→시각적 undo). 접을 수 있음.
- **중앙(문서 미리보기)**: **실제 문서를 HTML로 렌더**(`export_document`/kordoc `renderHtml`). [원본 / 변경본 / diff] 탭. 변경 셀·문단을 하이라이트.
- **우(대화)**: 스트리밍 채팅 + 툴콜 활동(접이식) + **빠른 작업 버튼**(요약·검토·개인정보 가리기·내보내기).
- **상단바**: 작업 폴더, 모델 전환, 컨텍스트 게이지.
- **하단 Composer**: 자연어 입력 + 첨부.

## 4. CLI 기능 → GUI UX 매핑 (구현된 20개 도구 기준)

| CLI 도구/기능 | GUI 표현 |
|---|---|
| `read_document`(요약·검토·검색) | 드롭/클릭 시 자동 읽기 → 중앙 패널 렌더 + 우측에 "이 문서로 무엇을 할까요?" 제안 칩 |
| `find_in_document` | 미리보기 내 **검색창 → 클릭으로 위치 점프**(표 셀이면 좌표 칩, 본문이면 스크롤) |
| `propose_edit` | 승인 다이얼로그: 텍스트 **인라인/병렬 diff**(추가=초록·삭제=빨강) |
| `propose_find_replace` | 매치별 미리보기 카드(전/후) + "N곳 변경" 배지. 서식 분리 텍스트도 처리됨을 표시 |
| `propose_cell_edit` | 표를 그려 **바뀌는 셀만 하이라이트**(전→후), 좌표/라벨 모드 칩 |
| `propose_table_structure` | 행·열 추가/삭제를 표 위에 **고스트 행/열**로 미리보기 |
| `propose_form_fill` / `extractFormSchema` | **양식 폼 UI** — 라벨·타입(날짜/전화/금액)별 입력칸, 필수 표시. 미매칭 라벨은 "채울 수 있는 필드" 제안 |
| `list_form_objects` / `propose_form_object` | 편집상자·체크박스·콤보를 **위젯으로** 렌더해 값 입력 |
| `scan_pii` / `propose_redact_pii` | 문서에서 **PII를 형광 표시** → "가리기" 토글, 마스킹 전/후 미리보기 |
| `export_document` | 우하단 **⇩ 내보내기**(HTML/PDF). PDF 미가용 시 안내 |
| `write_new_document` / `write_new_spreadsheet` | "새 문서" 버튼 → 내용 미리보기 후 저장 |
| `compare_documents` | 두 파일 드롭 → **신구대조표** 시각화 |
| `list_backups` / `restore_backup` | 좌측 **되돌리기 타임라인**(시각/요약) → 클릭 복원(복원도 승인) |
| 한국 법령 MCP(`korean-law`) | 검토 결과에 **법령 인용 카드**(조문·현행 여부 배지·출처 링크) |

## 5. 승인(Approval) UX — 신뢰의 핵심

GUI의 가장 중요한 화면. `ApprovalHandler`(core 공용)가 보내는 `Proposal`을 **종류(kind)별 맞춤 렌더**:

- 공통 헤더: **요약 · 대상 파일 · ⚠ 경고(있으면) · 포맷 변환 안내**(예: `.hwp→.hwpx`).
- `edit`/`find-replace`: 인라인 텍스트 diff.
- `cell-edit`/`table-structure`: 표 시각 diff(셀·행·열 하이라이트).
- `redact-pii`: 마스킹 하이라이트(원문 값은 **표시하지 않음** — CLI와 동일 정직성).
- `form-fill`/`form-object`: 폼 입력 결과 표.
- `export`: 출력 형식·파일명·미리보기.
- `restore`(되돌리기): 복원 시점·요약.
- 액션: **[승인] [거절]**. (향후: per-hunk 부분 승인.)
- 항상 보이는 안전 문구: *"승인 전까지 원본은 바뀌지 않습니다."*

배경 자동화(사용자에겐 은은하게): **자가검증 라운드**("검증 중…" → "검증 완료"), **thrash 억제**(반복 헛호출 시 자동 전략 전환), **능력·한계 grounding**(불가 작업은 회색 안내).

### 5.1 변경 미리보기·diff 전략 — **rhwp 비의존(확정)**

미리보기 렌더링에 **rhwp(Rust+WASM)를 쓰지 않는다.** rhwp는 개발 중·불안정(과거 저장 손상 이력)이고 0.6.0에서 의존성에서 제거됐다 — 이를 GUI에서 되살리면 상류 변경에 휘둘린다. 대신 **git diff 스타일 2층** + 최종은 한글 확인:

1. **변경 내역(git diff 스타일) — 정밀·결정적**: 소스는 각 도구가 이미 만드는 `proposal.diff`(`parse → markdown` 기반). 텍스트=±라인 색상, 셀=전→후 표, PII=마스킹 하이라이트. **렌더 엔진 불필요** → 결정적·안정. "정확히 무엇이 바뀌는지"를 책임진다.
2. **before/after 시각 컨텍스트(HTML) — 근사**: `renderHtml(원본 md)` vs `renderHtml(편집 md)`를 **나란히(split)** 또는 토글. 바뀐 텍스트/셀을 양쪽에 하이라이트(remove=빨강 / add=초록). 순수 함수 `renderHtml`만 사용(엔진 0). *G4에서 변경 영역 인라인 하이라이트로 고도화.*
3. **최종 진실 = 한글에서 열기(승인 후)**: HTML 렌더는 **근사**임을 정직히 표시 — 픽셀 정확은 사용자가 한글에서 확인(한컴 육안검증 원칙과 동일).

→ 미리보기 의존성은 kordoc `parse`+`renderHtml`(둘 다 안정)뿐. 정확성은 텍스트 diff가, 시각 맥락은 HTML이 담당하며, **렌더 엔진 변경 리스크가 GUI에 전파되지 않는다.**

## 6. 온보딩·편의 (비전문가 배려)

- **첫 실행 설정 마법사**: API 키(Claude/OpenAI/Gemini) 입력, (선택) 한국 법령 키 발급 안내 — CLI `config set`을 GUI 폼으로.
- **드래그&드롭이 출발점**: 파일을 끌어다 놓으면 자동 읽기 + 추천 작업 칩.
- **빠른 작업 버튼**: 요약 · 검토(법령) · 개인정보 가리기 · 내보내기 · 되돌리기.
- **명령 동등성**: CLI `/usage`(누적 토큰)·`/context`·`/model`·`/clear`를 상태바/메뉴 위젯으로.
- **안심 신호**: 매 변경에 자동 백업 → 되돌리기 타임라인. "원본 무변경" 인디케이터.
- **포맷 정직 안내**: `.hwp` 표·셀 편집 불가 등은 변환 버튼(".hwpx로 저장")과 함께 안내.

## 7. 기술 구조

- **공유 엔진**: `@kodocagent/core`(AgentSession·ToolRegistry·ApprovalHandler·SessionStore) + `@kodocagent/doc-tools`(20개 도구) + `@kodocagent/shared`. **GUI는 이를 import만** 한다(로직 중복 금지).
- **메인 프로세스**(`agent-bridge.ts`): AgentSession 실행, 이벤트→IPC, 승인 왕복, 파일 시스템(작업 폴더·드롭).
- **프리로드**: `window.kodoc.*` 안전 브릿지(chat.send/onEvent, approval.respond, config, cwd).
- **렌더러**(React): 위 패널들. 문서 미리보기는 kordoc `renderHtml`(메인에서 생성→IPC) 또는 `export_document` 결과를 iframe/샌드박스 렌더.
- **보안**: contextIsolation·nodeIntegration off, IPC 화이트리스트. 파일 접근은 작업 폴더 경계(core `resolveSafePath`)와 동일 원칙.

## 8. 단계적 로드맵 (제안)

- **G1 — 기본 워크스페이스**: 현 스캐폴드 + 드래그&드롭 열기 + **문서 HTML 미리보기**(renderHtml) + 상태바(모델·컨텍스트·토큰). *가장 큰 체감 가치.*
- **G2 — 시각 승인**: kind별 승인 다이얼로그(텍스트 diff·표 diff·PII 하이라이트·폼). 되돌리기 타임라인.
- **G3 — 빠른 작업·온보딩**: 추천 칩, 빠른 작업 버튼, 설정 마법사, 법령 인용 카드.
- **G4 — 고도화**: per-hunk 승인, 변경 전/후 양면 미리보기, 내보내기(PDF 의존성), 멀티 문서 비교.

## 9. 비고

- GUI는 CLI 기능의 **상위집합이 아니라 시각화 레이어**다. 새 편집 능력은 CLI(core/doc-tools)에 먼저 구현하고 GUI가 렌더한다.
- 미지원 하드티어(스타일·머리말/꼬리말·각주·상호참조·이미지, `.hwp` 표 셀, 중첩표 셀)는 GUI에서도 **회색 처리 + 정직 안내**로 일관 유지. (중첩표 셀은 scanSectionXml로 향후 구현 가능 — 구현되면 GUI도 자동 노출.)
- 상세 도구 동작·승인 모델은 [SPEC.md](SPEC.md), 로드맵은 [ROADMAP.md](ROADMAP.md) 참조.
